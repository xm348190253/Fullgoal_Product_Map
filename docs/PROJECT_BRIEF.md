# 项目说明

## 目标

基于 `Dunk0713/product-panorama` 的静态产品全景页面，生成富国基金版本。

## 数据口径

- 产品池来自天天基金公开 `fundcode_search.js`。
- 本次按基金名称以“富国”开头筛选，生成 812 条产品代码记录。
- 包含天天基金列表里的后端收费代码和 REIT 代码；页面默认只展示每个基金家族的主份额，勾选“显示份额变体”后展示全部代码。
- 业绩、规模、基金经理仍由页面运行时通过天天基金 / 东方财富公开接口拉取。

## 实现决定

- 保留原项目的单页静态结构，避免重构。
- 将产品池变量改为 `window.FULLGOAL_FUND_POOL`，页面逻辑同步读取该变量。
- 同类对比里的内部平替从“中欧”改为“富国”，市场竞品会排除富国产品。
- 本地缓存 key 从 `zo_*` 改为 `fg_*`，避免和原中欧版本缓存混用。
- 市场脉搏固定展示 9 个指数：上证指数、沪深300、创业板指、科创50、恒生指数、恒生科技、标普500、纳斯达克、道琼斯指数；窄屏显示不全时自动循环播放。若东方财富 push2 行情接口失败，则使用 A/H 2026-06-01 收盘值和美股最近收盘值兜底，避免显示 `--`。
- 页面交互必须统一作用于同一个当前结果池：一级 / 二级 / 三级筛选、搜索、自选、份额变体、排序、KPI、图表、表格和 CSV 导出都以当前前端状态为准。
- 筛选层级保持“一级大类 -> 二级策略 -> 三级细分”：切换一级时重置二级和三级，切换二级时重置三级；当前选中产品被筛掉时关闭详情抽屉并提示。
- 金融数据展示不得伪造收益率、规模、净值、回撤、评级或申购状态；缺失数据统一显示 `--`，错误反馈使用 Toast，不使用 `alert()`。
